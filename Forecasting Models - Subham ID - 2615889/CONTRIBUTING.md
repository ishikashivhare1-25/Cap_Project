# Contribution Guidelines

Thank you for your interest in contributing to the Demand Forecasting Model project!

## How to Contribute

### Reporting Bugs
- Check if the issue already exists
- Provide detailed description of the bug
- Include steps to reproduce
- Share error messages and logs

### Suggesting Enhancements
- Clearly describe the enhancement
- Explain why it would be useful
- Provide examples if applicable

### Submitting Changes

1. Fork the repository
2. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Test your changes thoroughly
5. Commit with clear messages:
   ```bash
   git commit -am 'Add: description of changes'
   ```
6. Push to your branch:
   ```bash
   git push origin feature/your-feature-name
   ```
7. Submit a Pull Request

## Code Style

- Follow PEP 8 conventions
- Use descriptive variable names
- Add comments for complex logic
- Include docstrings for functions

## Pull Request Process

1. Ensure your code doesn't break existing tests
2. Update documentation if needed
3. Reference any related issues
4. Write a clear PR description
5. Be responsive to feedback

## Getting Help

- Open an issue for questions
- Check existing issues and discussions
- Comment on related PRs for context

Thank you for contributing!
